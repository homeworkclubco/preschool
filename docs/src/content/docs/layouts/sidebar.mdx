A layout for placing two elements side-by-side. When space permits, the sidebar element has a set width and the content element takes up the remaining space. When space is constrained, elements collapse into a single column.

## Basic Usage

The sidebar layout requires two child elements - the first acts as the sidebar, the second as the main content.

```html
<div class="sidebar">
    <aside>Sidebar content</aside>
    <main>Main content</main>
</div>
```

## Default Behavior

- Left sidebar with content-based width (auto)
- Content element has 50% minimum width
- Elements stretch to equal height
- Collapses to single column when content minimum cannot be maintained

## Sidebar Width

Control the fixed width of the sidebar element using width modifiers:

```html
<div class="sidebar sidebar--side-xs">...</div>
<!-- ~240px - narrow nav -->
<div class="sidebar sidebar-width-sm">...</div>
<!-- ~320px - typical nav -->
<div class="sidebar sidebar--side-md">...</div>
<!-- ~400px - wide nav -->
<div class="sidebar sidebar--side-lg">...</div>
<!-- ~480px - dashboard panel -->
```

## Content Minimum Width

Set the minimum width for the content area to control when the layout collapses:

```html
<div class="sidebar sidebar--content-min-xs">...</div>
<!-- 40% - allows narrow content -->
<div class="sidebar sidebar--content-min-sm">...</div>
<!-- 50% - default, balanced -->
<div class="sidebar sidebar--content-min-md">...</div>
<!-- 60% - content priority -->
<div class="sidebar sidebar--content-min-lg">...</div>
<!-- 70% - heavy content focus -->
<div class="sidebar sidebar--content-min-prose">...</div>
<!-- var(--measure) - optimal reading width -->
```

The `prose` variant is particularly useful when the main content is text-heavy, as it maintains optimal line length for readability.

## Sidebar Placement

By default, the sidebar appears on the left. Use `sidebar--right` to place it on the right:

```html
<div class="sidebar sidebar--right sidebar--side-md">
    <main>Main content (appears first/left)</main>
    <aside>Sidebar (appears second/right)</aside>
</div>
```

## Prevent Equal Height

By default, both elements stretch to match the tallest one. Use `sidebar--no-stretch` to prevent this:

```html
<div class="sidebar sidebar--no-stretch">
    <aside>Short sidebar</aside>
    <main>Tall content...</main>
</div>
```

## Common Examples

### Documentation Layout

```html

    class="sidebar sidebar-width-sm sidebar--content-min-prose sidebar-space-lg"
>
    <nav>
        <h2>Navigation</h2>
        <ul>
            <li><a href="#intro">Introduction</a></li>

            <li><a href="#api">API</a></li>

        <h1>Documentation</h1>
```

## Customization

Override sidebar behavior with CSS custom properties:

```css
.sidebar {
    /* Control sidebar width */
    --sidebar-side-width: 18rem;

    /* Control content minimum width */
    --sidebar--content-min: 65%;

    /* Control gap between elements */
    --sidebar-space: var(--spacing-md);
}
```

Or create custom width modifiers:

```css
.sidebar-width-custom {
    --sidebar-side-width: 22rem;
}
```

## Accessibility

The sidebar layout is purely presentational. Ensure proper semantic HTML:

```html
<div class="sidebar">
    <nav aria-label="Main navigation">
        <!-- Navigation links -->
    </nav>
    <main>
        <!-- Primary content -->
    </main>
</div>
```

- Use semantic elements (`<nav>`, `<aside>`, `<main>`)
- Provide appropriate ARIA labels when needed
- Ensure content order makes sense when layout collapses
- Test with keyboard navigation and screen readers

## How It Works

The sidebar layout uses flexbox with the following technique:

1. The sidebar has a fixed `flex-basis` (or auto for content-based)
2. The content has `flex-basis: 0` and `flex-grow: 999` to consume available space
3. The content has a `min-inline-size` that triggers wrapping when space is insufficient
4. When wrapped, both elements take full width due to `flex-wrap: wrap`

This creates a responsive layout without media queries, adapting to the container width automatically.
