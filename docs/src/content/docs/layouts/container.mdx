A centered layout container that constrains content width and provides consistent horizontal padding. The container automatically centers itself and adapts to different content types with preset width options.

## Basic Usage

The container centers content with a default max-width of 110rem (~1760px) and responsive horizontal padding.

```html
<div class="container">
    <h1>Centered Content</h1>
    <p>This content is constrained to a maximum width and centered on the page.</p>
</div>
```

<Iframe>
    <div class="container" style="border: 1px solid #ccc; background: #f0f0f0;">
        <div style="background: #e8e8e8; padding: 1rem;">
            <h3>Container Content</h3>
            <p>Centered with max-width of 110rem and auto margins</p>
        </div>
    </div>
</Iframe>

## Default Behavior

- Max-width: 110rem (~1760px)
- Centered with `margin-inline: auto`
- Horizontal padding: `var(--space-md-lg)` (responsive)
- Padding adjusts with viewport size


## Full-Bleed Layouts

Full-bleed containers create edge-to-edge layouts with a 12-column grid system, perfect for breaking out of the container while maintaining alignment.

### Full-Bleed 12-Column

```html
<div class="container container--full-bleed-12">
    <div style="grid-column: 1 / -1;">Edge to edge</div>
    <div style="grid-column: 2 / -2;">Content with gutters</div>
</div>
```

The full-bleed container:

- Removes padding
- Creates a 12-column grid with outer gutters
- Each column is `max-width / 12`
- Outer columns (`1` and `-1`) provide edge-to-edge capability

### Full-Bleed 6-Column

```html
<div class="container container--full-bleed-6">
    <div style="grid-column: 1 / 7;">Left half</div>
    <div style="grid-column: 7 / -1;">Right half</div>
</div>
```

**Note:** The current implementation sets `--container-columns: 12` for both variants. You may want to adjust this based on your needs.

## Customization

Override container behavior with CSS custom properties:

```css
.container {
    /* Change default max-width */
    --container-max-width: 90rem;

    /* Adjust horizontal padding */
    --container-padding: var(--space-xl);
}
```

Create custom width variants:

```css
.container.is-ultrawide {
    --container-max-width: 140rem;
}

.container.is-reading {
    --container-max-width: 68ch; /* Character-based width */
}
```

Adjust full-bleed grid columns:

```css
.container.container--full-bleed-custom {
    --container-columns: 16; /* 16-column grid */
}
```

## Responsive Padding

The default padding (`--space-md-lg`) is responsive and adjusts based on viewport size. This ensures appropriate spacing on both mobile and desktop devices.

To override with custom responsive padding:

```css
.container {
    --container-padding: clamp(1rem, 4vw, 3rem);
}
```

## Accessibility

The container is purely presentational and doesn't affect semantics:

```html
<div class="container container--prose">
    <main>
        <h1>Main Content</h1>
        <!-- Use semantic HTML as normal -->
    </main>
</div>
```

- Container doesn't affect heading hierarchy
- Doesn't interfere with screen reader navigation
- Maintains document outline
- Use semantic elements (`<main>`, `<article>`, `<section>`) as appropriate

## How It Works

The container uses modern CSS for centering and responsive spacing:

### Standard Container

1. `max-width` constrains content width
2. `margin-inline: auto` centers horizontally
3. `padding-inline` adds horizontal padding that adapts responsively

### Full-Bleed Grid

1. Creates a 14-column grid (12 content + 2 gutters)
2. `minmax(0, 1fr)` for outer gutters allows edge-to-edge content
3. Inner columns use `minmax(0, calc(max-width / columns))` for proportional sizing
4. Content can span any columns using `grid-column`

## Comparison with Other Layouts

- **Container**: Constrains width, centers content, adds padding
- **Stack**: Vertical spacing between children
- **Sidebar**: Side-by-side layout with flexible sidebar
- **Cluster**: Horizontal wrapping layout with gaps

Use container as the outer wrapper, then use other layout primitives inside for content organization.
