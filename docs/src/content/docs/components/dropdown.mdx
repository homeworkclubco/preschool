A framework-agnostic dropdown menu component for interactive menus.

## Basic Usage

```html
<hc-dropdown>
    <button slot="trigger" class="btn">Open Menu</button>
    <div slot="content">
        <a href="#">Option 1</a>
        <a href="#">Option 2</a>
        <a href="#">Option 3</a>
    </div>
</hc-dropdown>
```

## How It Works

The dropdown uses two slots:

- `trigger` - The button or element that opens the dropdown
- `content` - The menu content

## Features

- **Click outside to close** - Automatically closes when clicking outside
- **Escape key support** - Press Escape to close
- **Toggle behavior** - Click trigger again to close
- **Custom events** - Dispatches `hc-dropdown-open` and `hc-dropdown-close` events
- **Framework-agnostic** - Works with any framework or vanilla HTML

## Listening to Events

```js
const dropdown = document.querySelector('hc-dropdown');

dropdown.addEventListener('hc-dropdown-open', () => {
    console.log('Dropdown opened');
});

dropdown.addEventListener('hc-dropdown-close', () => {
    console.log('Dropdown closed');
});
```

## Programmatic Control

Control the dropdown with JavaScript:

```js
const dropdown = document.querySelector('hc-dropdown');

// Check if open
console.log(dropdown.open); // true or false

// Open
dropdown.open = true;

// Close
dropdown.open = false;

// Toggle
dropdown.open = !dropdown.open;
```

## Customization

The dropdown respects CSS variables from your color scheme:

```css
hc-dropdown {
    /* Adjust positioning */
    --space-2xs: 0.5rem;

    /* Change appearance */
    --color-bg-alt: your-color;
    --color-border: your-border-color;
    --shadow-lg: your-shadow;
}
```

## Accessibility

The dropdown component includes:

- `role="menu"` on content
- Focus management
- Keyboard navigation (Escape to close)
- Click outside to close

For better accessibility, consider:

- Adding `aria-label` to icon-only triggers
- Using semantic navigation elements
- Ensuring sufficient color contrast
